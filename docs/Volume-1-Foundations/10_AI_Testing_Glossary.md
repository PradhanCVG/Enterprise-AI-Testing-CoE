# AI Testing Glossary

## Overview

This glossary defines common terms used in enterprise AI quality engineering and validation programs.

## Key Terms

- Drift: A measurable change in data, model behavior, or outcomes over time.
- Groundedness: The degree to which a generated answer is supported by provided context.
- Hallucination: Content generated by an AI system that is unsupported or fabricated.
- Guardrails: Controls that constrain unsafe, non-compliant, or undesired behavior.
- Evaluation Set: A curated dataset used to measure model or system quality.
- Traceability: The ability to link requirements, data, tests, results, and approvals.
